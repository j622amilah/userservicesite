Reporting Data Science Information Page

Presenting Data Science results in a way that is easy to understand is important for data insights to be realized. This page explores different ways to report information.

[0] Professional Blogging

The first and most noteable way to report information is to present clear guides/tutorials using professional Data Science blogging.

[1] Machine Learning/Deep Learning webapps (more complex models in progress!!)

Seeing a model predict a result directly from data allows for others to better understand their data, thus allowing for data insights to be found!
  • Text category prediction webapp (out of service streamlit webapp)

[2] Google Apps Scripts webapps (more to come!)

Seeing data summarized in a figure that updates dynamically with respect to the data allows for others to summarize and understand their data, thus allowing for data insights to be found!
  • Google BigQuery Connected Sheet Dashboard with Google Apps Scripts (GET request webapp) 📄

[3] Formatted Case Study Reports

A classical reporting format of formal Data Analytics projects. It is important to have a document clearly explaining the goals, expectations/requirements, and measurable deliverables of success for a project, in order to have a stable, calm, and productive working environment. These case study reports using Google Analytics organization are formatted such that clear, logical, and understandable projects are created and executed with team success (even if the results are less desirable); working as a team and following logical short-term and long-term goals creates eventual success.
  • A collection of Data Analytics Case Studies using Google Analytics/Business Intelligence organization
